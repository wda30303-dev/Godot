# AGENTS.md

本文件是此 Godot 專案的 agent 工作守則。

AI 每次進入此工作區工作前，必須先閱讀本文件，並依照這裡定義的規則行動。本文件不是補充資料，而是定義 agent 在此工作區中的行為規則：要讀哪些檔案、怎麼更新記憶、哪些事可以直接做、哪些事必須先詢問使用者。

穩定的專案資訊、工作紀錄、設計決策、待確認問題，請依照規則記錄到 `.agent_memory/` 內的對應檔案。

## 開始工作前的讀取規則

每次開始工作前，必須先閱讀：

1. `AGENTS.md`

其他記憶檔不要求每次全部讀完，應依照任務內容、影響範圍與風險自行判斷是否需要讀取。使用者或系統若指定要參考某份檔案，則優先讀取該檔案。

可依情況讀取：

- `.agent_memory/PROJECT_CONTEXT.md`：需要理解專案目前穩定狀態時閱讀。
- `.agent_memory/IMPLEMENTATION_RULES.md`：準備修改程式、場景、系統架構時閱讀。
- `.agent_memory/DESIGN_DECISIONS.md`：需要確認已定案設計時閱讀。
- `.agent_memory/OPEN_QUESTIONS.md`：遇到未定案規則、需要詢問使用者前閱讀。
- `.agent_memory/FEATURE_BACKLOG.md`：討論或實作未來功能時閱讀。
- `.agent_memory/WORK_LOG.md`：需要了解近期修改紀錄時閱讀。

## 需要讀完整記憶的情況

若本次工作符合以下任一情況，應讀取所有相關記憶檔，必要時讀完整 `.agent_memory/`：

- 修改核心玩法。
- 新增或修改跨系統功能。
- 涉及猜數字、點數、技能、士氣、單位、節點、防禦建築之間的互動。
- 需要決定長期設計方向。
- 使用者要求整理規劃、制定架構或建立新系統。
- 前一次工作留下未完成狀態，需要接續處理。

## 輕量工作可只讀必要檔案

若只是小範圍修改，不需要強制讀完整記憶。例如：

- 修一個明確 bug。
- 改一段 UI 文字。
- 補一個簡單註解。
- 查看單一檔案的行為。
- 回答使用者對目前某段程式的問題。

這類任務可只讀 `AGENTS.md` 與直接相關的 `.gd`、`.tscn` 或記憶檔。

## 修改前仍需理解相關檔案

若使用者要求新增、修改、修正功能，必須閱讀與該功能直接相關的 `.gd`、`.tscn` 或資源檔後再動手。

不要只靠檔名或先前印象修改程式。

## 工作語言

- 預設使用繁體中文回覆。
- UI 文字、註解、設計紀錄優先使用繁體中文。
- 現有專案內有部分中文亂碼。判斷功能時，以程式行為為準。
- 修正亂碼時，不要順手改動無關邏輯。

## 可以直接做的事

agent 可以直接進行：

- 閱讀專案檔案。
- 分析目前遊戲設計。
- 整理文件草稿。
- 修改使用者明確指定的功能。
- 修正明確的 bug。
- 新增小範圍、可回復、符合現有設計的輔助檔案。
- 在有修改檔案時，更新 `.agent_memory/WORK_LOG.md` 記錄本次工作結果。

## 必須先詢問使用者的事

以下事情不可自行決定，必須先問：

- 改變核心玩法規則。
- 決定技能名稱、技能上限、升級費用、解鎖門檻。
- 決定 A/B/C 如何換算點數。
- 決定菁英單位與王單位的解鎖次數。
- 決定防禦建築種類、攻擊方式、是否可破壞。
- 大幅重構單位、戰鬥、猜數字、節點系統。
- 刪除既有檔案或移除既有功能。
- 修改尚未確認的長期設計方向。

## 每次修改檔案後必須更新記憶

只要本次工作有修改任何檔案，工作完成後就必須依情況更新 `.agent_memory/`：

- 專案穩定事實：更新 `PROJECT_CONTEXT.md`
- 本次做了什麼：更新 `WORK_LOG.md`
- 使用者確認過的設計決策：更新 `DESIGN_DECISIONS.md`
- 尚未確認的問題：更新 `OPEN_QUESTIONS.md`
- 實作規則或程式注意事項：更新 `IMPLEMENTATION_RULES.md`
- 未來功能清單：更新 `FEATURE_BACKLOG.md`

若只是回答問題、討論方向、沒有修改檔案，也沒有新增穩定資訊，則不必更新記憶。

## 修改原則

- 先理解現有場景與腳本，再修改。
- 小步修改，避免一次重構太多系統。
- 保留目前原型行為，除非使用者要求替換。
- 猜數字、技能、點數、士氣、單位、節點、防禦建築應盡量分成不同系統。
- 未確認的設計只寫成可調整資料，不要硬編碼。
- 新增功能後，盡可能執行 Godot 專案或做針對性的腳本檢查。
- 若無法執行 Godot，也要在回覆中說明哪些部分已檢查、哪些尚未驗證。
